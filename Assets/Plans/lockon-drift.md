# プロジェクト概要
- **ゲームタイトル**: オフロード・カー・シューティング プロトタイプ (仮)
- **ハイレベルコンセプト**: 武装したオフロードカーを操作し、動的に無限生成されるローポリでフラットシェーディングされた美しい3Dのフィールドを縦横無尽に駆け巡りながら、ターゲットを撃破していく高速アーケード車両シューティングゲーム。
- **プレイヤー数**: シングルプレイヤー（キーボード＋マウス操作）
- **インスピレーション・参考ゲーム**: 『Smuggler's Run』、『Desert Strike』
- **トーン / アートディレクション**: スタイリッシュで鮮やか、クッキリとしたシャドウとフラットシェーディング（面ごとにカクカクした陰影が出る手法）によるローポリゴン世界。
- **対象プラットフォーム**: PC（スタンドアロン / WebGL）
- **画面の向き / 解像度**: 横画面 1920x1080
- **レンダーパイプライン**: Universal Render Pipeline (URP)

# ゲームメカニクス
## コアゲームプレイループ
プレイヤーはエンドレスなローポリ地形を進み、`E` キー（トグル）によって前方に存在する最も近いターゲットダミーをロックオンします。
ロックオン中、自車はターゲットを自動射撃しながら、ターゲットを中心とした軌道上を旋回する、爽快な「周回ドリフト」アクションを展開します。

## コントロールと操作方法（ロックオン＆ドリフトに特化した操作）
- **移動（通常時）**: `W`/`S` で前進・後退アクセル、`A`/`D` でハンドル旋回。
- **ロックオン（新機能）**: `E` キー（トグル操作）。前方の視界内に存在する最も近いターゲットに照準を固定。
- **射撃**: 手動の射撃ボタン（従来のSpaceやJキー）は**完全に削除**。ロックオン成立中のみ、**完全自動射撃**が作動します。
- **ロックオン中のドリフト挙動（W/S, A/Dの仕様変更）**:
  - 車両の機首（正面）は、常にターゲットの方向を強制的に注視（LookAt）します。
  - **`W` / `S` キー (公転速度の増減)**: 
    ターゲットを中心とする円軌道の**接線方向（Tangent）の力**を制御します。`W` を押すと公転が加速（周回移動速度の増加）し、`S` を押すと減速・逆回転します。
  - **`A` / `D` キー (ロックオン軸からの方向オフセット / 軌道半径の制御)**:
    ターゲットと自車を結ぶ**法線方向（Radial）の力**を制御します。`A` キーでターゲットへ内側にスライド（軌道半径を縮小）、`D` キーで遠ざかる方向へ外側にスライド（軌道半径を拡大）します。
  - 車両本体がターゲットを向きながら、接線（W/S）と法線（A/D）の物理的な推進力で移動するため、**非常にダイナミックな横滑り「軌道周回ドリフト」**がキー入力に応じて発生します。

# 実装プラン（手順）

### ステップ 1: ロックオン対象の探索・切り替えの実装 (`OffroadCarController.cs`)
- `E` キーの入力を検知（トグル）。
- ロックオン開始時、車の前方120度のコーン、または画面内かつ一定の最大射程距離（例: 80m）の中に存在するアクティブな `TargetDummy` コンポーネントを探索。
- 該当するターゲットのうち、最も距離が近いもの、あるいは視線の中心に近いものを `lockedTarget` として設定。
- 再度 `E` キーを押す、あるいは対象が破壊・アンロード（消滅）した場合はロックオンを解除。

### ステップ 2: ロックオン中の「自動射撃」の実装 (`CarWeapon.cs`)
- `CarWeapon.cs` にあった `Space` / `J` キーの入力チェックコードを完全に削除。
- `OffroadCarController` が保持する `lockedTarget` の参照を確認。
- ターゲットが非 null であれば、以下を毎フレーム実行：
  1. 砲塔（`turretPivot`）の向きを、マウス追従ではなくターゲットの座標に向けて滑らかに旋回・固定。
  2. `Time.time >= nextFireTime` であれば、弾丸を自動的に射撃（`FireBullet()`）。

### ステップ 3: 「軌道ドリフト（周回物理）」の実装 (`OffroadCarController.cs`)
ロックオン中の移動制御を、ターゲットを中心とした物理的な円軌道に拘束します。

#### 📐 ドリフトの物理挙動モデル：
1. **車体姿勢の固定**:
   - `transform.LookAt(targetPosition)` もしくはそれに近いトルクを加え、車の正面を常にターゲットに向けます（これによって、常にターゲットを正面に据えたまま横滑り（ドリフト）する姿勢になります）。
2. **`W` / `S` キーによる公転速度の増減（接線フォース）**:
   - 自車 $P$ からターゲット $C$ へのベクトル $d = P - C$ を正規化し、XZ平面での直交接線ベクトル $t = (-d.z, 0, d.x)$ を求めます。
   - `W` キーが押されたとき $t$ 方向に推進力を、`S` キーで $-t$ 方向に推進力を加えます。これにより、円周上のスピードが加速・減速・逆転します。
3. **`A` / `D` キーによる軌道オフセット（法線フォース）**:
   - 正規化した方向ベクトル $u = d / |d|$（ターゲットから外側へ向かうベクトル）を用います。
   - `A` キー（内側オフセット）が押されたとき $-u$ 方向へ推進力を加え、ターゲットに近づきます。
   - `D` キー（外側オフセット）が押されたとき $u$ 方向へ推進力を加え、ターゲットから遠ざかります。
4. **進入時の慣性の維持**:
   - ロックオン開始時の速度ベクトルのうち、接線方向の運動量を一時的に維持し、ロックオンした瞬間に自然な「スライディング横滑り（進入慣性）」が発生するように初期インパルスを掛けます。

### ステップ 4: ロックオン中の対峙シネマティックカメラの実装 (`CarCameraFollow.cs`)
- ロックオン中、カメラは自車の真後ろに張り付くのではなく、「自車とターゲットの中間、あるいは車両から少し斜め後ろに引いた位置」から両方を1画面に収めるように位置を変更。
- 車体がターゲットの周囲を公転するダイナミックなアクションを、まるでアクション映画のカットのように美しく捉えます。

# 検証 & テスト計画
1. **ロックオン判定テスト**:
   - 広大な起伏のある平野に散ラバる複数の動的ターゲット dummy の前で `E` キーを押し、最も近い敵にレティクルが吸いつくか。破壊された際に即座にロックが解除されるか。
2. **自動射撃テスト**:
   - ロックオン中、手動操作なしで砲塔が自動で敵を捉え、正確にオレンジ色の弾丸を連射し、デブリの爆発が起きるか。手動射撃が完全に機能しなくなっているか。
3. **軌道ドリフトコントロール（物理挙動）の確認**:
   - `E` を押した状態で `W`/`S` を押したとき、車輪の摩擦に抗って綺麗な円を描く「接線公転加速」が発生するか。
   - `A`/`D` によって、ターゲットに近づいたり離れたりする「径方向のオフセット（軌道半径の増減）」が物理的にスムーズに（吹き飛んだりせず）実行できるか。
   - ロックオン解除（`E` を再押し）した瞬間に、公転の接線方向に向けてバギーが物理的慣性を保ったまま「慣性スリップ」で解き放たれるか。
