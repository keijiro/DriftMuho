# 📝 実装計画書: 巡回型敵ドローンの追加とプレイヤー体力・ゲームオーバーシステム

この実装計画書は、既存の静的標的を、`Drone.glb` モデルを使用した自律的に巡回・攻撃する敵ドローンに置き換え、プレイヤーの耐久力（HP）管理システム、およびサイバーパンク風のHUD表示とゲームオーバー演出を実装するためのステップを定義します。

---

## # プロジェクト概要

*   **ゲームタイトル**: オフロードカー・エンドレス・シューティング
*   **ゲームコンセプト**: URPとUI Toolkitをベースにした、ローポリゴンのオフロードバギーによる無限地形シューティングゲーム。プレイヤーはターゲット（敵ドローン）の周囲を物理挙動で高速ドリフト旋回（軌道周回）しながら、敵の激しい銃撃をかいくぐって戦います。
*   **プレイヤー**: シングルプレイヤー
*   **参考/インスピレーションゲーム**: Motorstorm, Borderlands, Mad Max
*   **トーン/アートディレクション**: サイバーパンク、レトロ・ローポリ、フラットシェード地形
*   **ターゲットプラットフォーム**: PC / WebGL
*   **画面解像度**: 横画面 (1920x1080)
*   **レンダーパイプライン**: Universal Render Pipeline (URP)

---

## # ゲームメカニクス

### コアゲームプレイループ
1.  **走行と探索**: 自動生成されるローポリ無限地形をバギーで快走します。
2.  **交戦と回避**: 浮遊する敵ドローンを発見したら、`E` キーでロックオン。ConfigurableJointによる高速サークルドリフトで回避軌道を描きながら、自動で弾を撃ってドローンを破壊します。
3.  **生存（ダメージ管理）**: 敵ドローンはプレイヤーを検知するとオレンジ色の球体弾で反撃してきます。これを食らうと「車体整合性（Vehicle Integrity）」が減少し、限界に達するとバギーが爆発大破します。
4.  **ゲームオーバーと再起動**: 車体が破壊されると、画面全体に「CRITICAL SYSTEM FAILURE」エラーが表示されます。`R` キーを押すと即座にシステム（シーン）が再起動し、新たなランでリトライできます。

### 操作方法
*   **アクセル/バック**: `W/S` または `上/下 矢印キー`
*   **ステアリング/軌道半径伸縮**: `A/D` または `左/右 矢印キー`
*   **ロックオン**: `E` キー（最も近いアクティブなドローンをロックオン）
*   **ハンドブレーキ**: `Space` キー
*   **システム再起動**: `R` キー（ゲームオーバー画面時のみ有効）

## # UI (ユーザーインターフェース)

### 1. 車体整合性ゲージ（ヘルスバー）
*   **配置場所**: 画面左上 (Top-Left)
*   **デザイン**: 深いグレーの半透明背景、ネオンレッド/オレンジのシャープな枠線、ヘルスが低下すると赤く点滅する警告インジケータ。
*   **要素構成**:
    *   `health-container`: ゲージ全体を囲むパネル。
    *   `health-bar-bg`: 最大耐久力を表す背景スロット。
    *   `health-bar-fill`: 現在の耐久力に合わせて横方向に伸縮する輝く赤いバー。
    *   `health-text`: 「100%」または数値（例: `100 / 100`）を示すテキスト。

### 2. システム大破エラー（ゲームオーバー画面）
*   **配置場所**: 画面中央フルスクリーン
*   **デザイン**: 周辺が暗くなるビネット効果、巨大な赤の警告タイトル、明滅する操作ガイド。
*   **要素構成**:
    *   `game-over-screen`: フルスクリーンオーバーレイ。
    *   `failure-title`: 「VEHICLE DESTROYED」（大破、太字赤）
    *   `failure-subtitle`: 「CRITICAL SYSTEM FAILURE」（深刻なシステム障害、白の等幅フォント）
    *   `reboot-guidance`: 「PRESS [R] TO REBOOT SYSTEM」（再起動ガイダンス、オレンジ黄色で明滅）

## # 主要アセット & コンテキスト

### 🛠️ 新規追加するスクリプト

#### 1. `Assets/Scripts/PlayerHealth.cs`
プレイヤーのバギーの耐久力を管理し、被弾時のダメージ処理、大破時の爆発エフェクト生成、ゲームオーバー画面のトリガー、Rキーによる再起動を受け持ちます。
*   **プロパティ**:
    *   `maxHealth` (float, 初期値 `100f`): 最大耐久力。
    *   `currentHealth` (float): 現在の耐久力。
    *   `isDead` (bool): 死亡フラグ。
*   **主要メソッド**:
    *   `TakeDamage(float damage)`: 耐久力を減らし、被弾フィードバックを実行。
    *   `Die()`: 車体を無効化し、キューブやオレンジ火花による大破爆発を生成。ゲームオーバーUIを起動。
    *   `Update()`: 死亡時に `R` キーが押されたらシーンを再ロード。

#### 2. `Assets/Scripts/EnemyDrone.cs`
敵ドローンのAI状態（巡回、攻撃）と挙動を管理します。
*   **プロパティ**:
    *   `hoverHeight` (float, 初期値 `2.5f`): 地面からの浮遊高度。
    *   `bobbingSpeed` (float, 初期値 `2.0f`), `bobbingAmount` (float, 初期値 `0.3f`): 浮遊時のゆらゆら動く揺れ幅。
    *   `patrolSpeed` (float, 初期値 `4.0f`): 巡回速度。
    *   `patrolRadius` (float, 初期値 `15f`): 生成起点からのランダム巡回半径。
    *   `detectionRange` (float, 初期値 `25f`): プレイヤー探知距離。
    *   `fireRate` (float, 初期値 `1.5f`): 射撃頻度（秒）。
*   **状態（States）**: `Patrol` (巡回), `Attack` (攻撃)
*   **主要ロジック**:
    *   `TerrainNoise.GetHeight(x, z)` を使用して、起伏の激しいローポリ地形に完全に沿って低空浮遊します！
    *   巡回中：起点周囲にランダムな目標地点（ウェイポイント）を定め、そこに向けてスムーズに向きを変えて移動。到着すると少し待機して次のウェイポイントへ。
    *   プレイヤー検知：範囲内に入ると攻撃状態へ移行。移動を緩め、プレイヤーの方向を向いてオレンジ色の弾を `fireRate` に応じて発射します。
    *   自身の `TargetDummy` の死亡確認：`TargetDummy.IsDead` が true になったら、全てのAI活動と弾の発射を完全に停止。

#### 3. `Assets/Scripts/EnemyBullet.cs`
敵が放つオレンジ色の球体プリミティブ弾の物理移動、生存期間、プレイヤーへの当たり判定を処理します。
*   **プロパティ**:
    *   `speed` (float, 初期値 `25f`)
    *   `damage` (float, 初期値 `15f`)
    *   `lifetime` (float, 初期値 `4.0f`)
*   **主要ロジック**:
    *   生成された方向（プレイヤーの方向）へ等速直線運動。
    *   `OnTriggerEnter` にて、プレイヤーコライダーへの接触を検知。
    *   プレイヤー衝突時：`PlayerHealth.TakeDamage()` を呼び出してダメージを与え、火花パーティクル（小さな立方体）を散らして自身を消滅させます。

### 📦 既存スクリプトの修正

#### 1. `Assets/Scripts/InfiniteTerrainManager.cs`
ドローンのGLBモデル（`Assets/Models/Drone.glb`）をシリアライズフィールドとして保持し、各チャンクの初期化時に受け渡します。
*   **追加点**:
    *   `[SerializeField] private GameObject dronePrefab;`
    *   `chunk.Initialize(...)` の引数に `dronePrefab` を追加。
    *   Editor上での自動割り当て（未設定時は `Assets/Models/Drone.glb` を自動ロード）。

#### 2. `Assets/Scripts/TerrainChunk.cs`
従来の仮プリミティブ（シリンダー＋球体）のターゲット生成処理を削除し、`dronePrefab` をロード・初期化する処理に変更します。
*   **修正点**:
    *   `Initialize` メソッドで `dronePrefab` を引数に受け取る。
    *   `SpawnTargets` メソッド内のプリミティブ作成（Cylinder/Sphere）を廃止。
    *   代わりに `dronePrefab` をインスタンス化し、そこに物理コライダー（SphereCollider/BoxCollider等）、Rigidbody、`TargetDummy`、`EnemyDrone` をアタッチ。
    *   ドローンが地形の起伏に沿って正しく配置されるように設定。

#### 3. `Assets/Scripts/VehicleHUD.cs`
UI Toolkit の UXML 要素にアクセスし、プレイヤーのHPおよびゲームオーバー画面の表示切替を制御します。
*   **追加点**:
    *   `health-bar-fill`、`health-text`、`game-over-screen` 要素を `Start()` 内でキャッシュ。
    *   `Update()` 内でプレイヤーの `PlayerHealth` を参照。
    *   耐久力パーセンテージに応じてバーの幅（`style.width`）とテキストをリアルタイム更新。
    *   大破（死亡）時に `game-over-screen` の `display` を `DisplayStyle.Flex` に切り替え。

#### 4. `Assets/UI/Main.uxml` & `Assets/UI/Main.uss`
新しいヘルスバー用のUIコンテナと、ゲームオーバー時のフルスクリーン赤警告シールドを追加・定義します。

## # 実装ステップ

### アプローチの選定理由
*   **アプローチA（統合コンポーネントモデル）を採用**: 既存の `TargetDummy`（ダメージ受け、大破爆発、Respawn）をそのままドローンオブジェクトのライフサイクル管理に再利用しつつ、自律行動（`EnemyDrone`）とプレイヤー耐久力（`PlayerHealth`）を個別モジュールとして分離します。既存の高度な物理ドリフト関節制御（`ConfigurableJoint`）や追従カメラへの悪影響を完全に排除しつつ、ロックオンシステムと完璧に統合します。

## # 検証 & テスト項目

1.  **スポーン確認とモデル適用（ビジュアルテスト）**:
    *   ゲームを開始し、地形の各チャンク内に静的プレースホルダーの代わりに、空中に浮かんだ `Drone.glb` の3Dモデルが美しくスポーンされていることを目視確認します。
    *   ドローンが起伏に沿って `2.5m` の高さをキープし、ふわふわと（サイン波で）上下に揺れながらランダムに巡回していることを確認します。
2.  **ロックオン＆旋回挙動（操作テスト）**:
    *   ドローンに近づいて `E` キーを押した際、従来通り武器タレットがドローンを自動追尾し、追従カメラがダイナミックな対峙構図（シネマティックモード）へブレンドされることを確認します。
    *   ドローンの周囲を高速でお尻を滑らせながら綺麗に物理ドリフト周回できるか確認します。
3.  **ドローンの索敵＆射撃（戦闘テスト）**:
    *   バギーがドローンの索敵範囲（25m）に侵入した際、ドローンがパトロールを止めて自車に向き直り、一定間隔でオレンジ色の球体弾を放ってくることを確認します。
    *   弾がバギーに当たった時にオレンジ色の火花（キューブデブリ）が散り、バギーがダメージを受けることを確認します。
4.  **耐久力ゲージの減少＆大破（HP/ゲームオーバーテスト）**:
    *   被弾するごとに、左上の「車体整合性」HPバーとパーセントテキストが減っていくことを確認します。
    *   HPが0になると、バギーが派手に爆発（立方体デブリや火花）して操作不能になり、画面中央に「VEHICLE DESTROYED - CRITICAL SYSTEM FAILURE」が大きく表示されることを確認します。
    *   その状態で `R` キーを押した際、シーンが即座にリロードされ、再び無傷の状態から無限地形でゲームが再開できることを確認します。
