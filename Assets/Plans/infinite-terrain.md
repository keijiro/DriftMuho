# プロジェクト概要
- **ゲームタイトル**: オフロード・カー・シューティング プロトタイプ (仮)
- **ハイレベルコンセプト**: 武装したオフロードカーを操作し、動的に無限生成されるローポリでフラットシェーディングされた美しい3Dのフィールドを縦横無尽に駆け巡りながら、ターゲットを撃破していく高速アーケード車両シューティングゲーム。
- **プレイヤー数**: シングルプレイヤー（キーボード＋マウス操作）
- **インスピレーション・参考ゲーム**: 『Smuggler's Run』、『Desert Strike』、その他クラシックなアーケードラリーゲーム。
- **トーン / アートディレクション**: スタイリッシュで鮮やか、クッキリとしたシャドウとフラットシェーディング（面ごとにカクカクした陰影が出る手法）によるローポリゴン世界。
- **対象プラットフォーム**: PC（スタンドアロン / WebGL）
- **画面の向き / 解像度**: 横画面 1920x1080
- **レンダーパイプライン**: Universal Render Pipeline (URP)

# ゲームメカニクス
## コアゲームプレイループ
プレイヤーは武装したオフロードカーを操縦し、自動で生成され続けるエンドレスなローポリ地形を進みます。車輪のスピード感とドリフト操作を楽しみつつ、地形の各所にランダム（または定期的）に出現するターゲットダミーを、マウス操作の旋回砲塔から放つ弾丸で狙い撃ち、爽快なエフェクトと共に破壊します。

## 操作方法と入力システム
- **ステアリング**: `A` / `D` キー (または `←` / `→` 矢印キー)
- **アクセル / バック**: `W` / `S` キー (または `↑` / `↓` 矢印キー)
- **砲塔のエイム**: 画面上のマウスカーソルが指す地面の座標を追従
- **射撃**: `Space` キー または `J` キー（長押しで自動連射）

# UI
ゲーム画面上に操作方法、現在の車両の絶対座標、現在の速度（km/h）、および現在生成されているチャンク（Chunk）数を表示するシンプルなHUDを追加。テンプレートに含まれる UI Toolkit（`Main.uxml`）の仕組みを拡張して実装します。

# 主要アセットと開発コンテキスト
1. **`Assets/Scripts/TerrainChunk.cs`** (新規コンポーネント)
   - 動的生成される個々のチャンク（Chunk）となる GameObject にアタッチされます。
   - 割り当てられた座標を基に、プロシージャル（手続き型）に3Dメッシュを生成。
   - グローバルなノイズ関数（世界座標依存）を使用して各頂点の高さを決定します。
   - 独自のメッシュ構築処理により、頂点を共有せず三角形ごとに独立させることで、**ローポリ特有の「フラットシェーディング（面ごとに影が分かれる見た目）」**を完全に実現します。
   - 車輪がめり込まず走行できるよう、生成したメッシュをリアルタイムに `MeshCollider` に設定します。

2. **`Assets/Scripts/InfiniteTerrainManager.cs`** (新規コンポーネント)
   - シーン内の空のマネージャーオブジェクトにアタッチされます。
   - 自車（PlayerCar）の現在位置を常に監視。
   - 自車がいる座標をチャンク単位のIDに変換：
     `currentChunk = (Mathf.RoundToInt(playerX / chunkSize), Mathf.RoundToInt(playerZ / chunkSize))`
   - プレイヤーを中心に指定された半径（例：周囲4チャンク、合計9x9グリッド）内の未生成チャンクを動的にスポーン。
   - プレイヤーから遠ざかったチャンクは自動で非アクティブ化、または破棄（プール）してメモリを最適化。
   - すべてのチャンクの繋ぎ目を完全に一致させるための、共通の高さ（ノイズ）計算処理を保持します。

3. **`Assets/Scripts/NoiseGenerator.cs`** または計算メソッド
   - 複数の周波数を重ね合わせるフラクタル・ブラウン運動（FBM）を用いたPerlinノイズ。
   - 世界座標（X, Z）のみをシードとして高さを一意に決定するため、チャンクの境界線上でも完全に高さが一致し、**継ぎ目が一切発生しません**。

4. **`Assets/UI/M_Floor.mat`**
   - 生成した地面に適用される URP Lit マテリアル。美しい陰影を表現できるように設定します。

# 実装ステップ（手順）

### ステップ 1: グローバル高さ計算ノイズの実装
- 世界座標を入力すると、一意の高さ（Y座標）を返す連続した関数を設計します：
  $$Height = f(X, Z)$$
- 周波数の異なる複数のPerlinノイズを重ね合わせることで、大きなうねりのある山と、細かいデコボコとしたオフロード感をブレンドします。

### ステップ 2: ローポリ・フラットシェードメッシュ生成クラス（`TerrainChunk.cs`）の実装
- チャンクは一定の大きさ（例：20m×20m）と解像度（例：16×16分割）で作成。
- 頂点ごとにグローバルノイズから Y 座標をサンプリング。
- **フラットシェーディング化の実装ロジック**:
  通常の共有頂点メッシュはスムーズな陰影（なめらかな丘）になります。面ごとの硬いエッジ（ローポリ感）を出すために、三角形ごとに独立した（頂点を共有しない）メッシュ配列に再構築します。
  1. 通常のグリッドからインデックスを取得。
  2. 頂点、UV、法線データを三角形の角ごとに独立して複製（3倍の頂点数）。
  3. `mesh.RecalculateNormals()` を呼び出し、境界の法線をブレンドさせないことで、きれいなカクカクした陰影を表現します。
- 完成したメッシュを `MeshFilter` と `MeshCollider` に即座に適用します。

### ステップ 3: 動的チャンク管理クラス（`InfiniteTerrainManager.cs`）の実装
- アクティブなチャンクを管理する `Dictionary<Vector2Int, TerrainChunk>` を定義。
- `Update()` 内で自車位置を毎フレーム（または数フレームおきに）チェック。
- 設定した表示半径（例：`viewRadius = 4`）の中にあって、辞書に存在しないチャンクを新しくインスタンス化。
- 表示半径より遠く離れた古いチャンクを非表示または削除してメモリを解放します。

### ステップ 4: 自車物理挙動およびシーンの更新
- シーン内にあった元々の静的オブジェクト `Floor` を無効化または削除します。
- `OffroadCarController` の接地判定（レイキャストなど）が、生成されたプロシージャルメッシュを正しく検出できるようにレイヤーまたは設定を調整します。
- 周囲のターゲットダミーが、起伏のある地面の上に自然に配置される（重力で着地する）ように調整します。

# 検証とテスト計画
1. **境界線の継ぎ目検証**:
   - 車を走らせてチャンク間をまたぐ際、視覚的な隙間や、物理的に車輪が引っかかる小さな段差がないかを厳密に確認。
2. **フラットシェード（見た目）の検証**:
   - ライトを当てたときに、ローポリらしいシャープな三角ポリゴンの面が美しく浮かび上がっているか、シーンビューとゲームビューで目視確認。
3. **無限生成の負荷・パフォーマンス検証**:
   - 車を特定の方向に数分間走らせ続け、チャンクの生成および削除時にフレームレートが著しく低下（ガタつき / スパイク）しないか確認。
4. **衝突判定の正確さ**:
   - 激しくバンプを乗り越えたり落下した際に、車が地面をすり抜けてしまわないことを実証。
